# DiagramCraft AI üöÄ

**DiagramCraft AI** is a powerful, full-stack application that uses Artificial Intelligence to instantly convert natural language descriptions into professional diagrams. Built with a modern tech stack, it leverages local LLMs (via Ollama) to ensure privacy and speed.

---

## üåü Project Overview

### **What does it do?**
You simply type a description like *"A user logs in, checks their balance, and transfers money"* or *"A database schema for an e-commerce app"*, and DiagramCraft AI automatically generates the corresponding **Mermaid.js** diagram code and renders it visually.

### **Key Features**
*   **ü§ñ AI-Powered Generation**: Uses LLaMA 3.1 (via Ollama) to understand complex requirements.
*   **üìä 10 Supported Diagram Types**:
    1.  **Flowchart**: Process flows and algorithms.
    2.  **Sequence**: System interactions and timelines.
    3.  **Class**: Object-oriented structures.
    4.  **State**: State machine transitions.
    5.  **ER Diagram**: Database schemas and relationships.
    6.  **Gantt**: Project schedules and timelines.
    7.  **Mindmap**: Brainstorming and hierarchy.
    8.  **Pie Chart**: Data distribution.
    9.  **User Journey**: User experience mapping.
    10. **GitGraph**: Version control workflows.
*   **‚úèÔ∏è Live Editor**:
    *   Real-time code editing.
    *   Instant visual updates.
    *   Immediate syntax error feedback.
*   **üíæ Export Options**:
    *   **PNG**: Download high-quality images for presentations.
    *   **MMD**: Download raw source code.
    *   **Copy**: One-click clipboard copy.
*   **üîê Secure Authentication**:
    *   User Registration & Login.
    *   JWT-based session management.
    *   Password hashing with Bcrypt.
*   **üìú History Tracking**: Automatically saves every generated diagram to your personal history.

## üõ†Ô∏è Technology Stack

*   **Python 3.11+**: Core programming language.

*   **MongoDB**: NoSQL database for storing users and diagram history.
*   **Pydantic**: Data validation and settings management.
*   **Passlib & Python-Jose**: Security and JWT handling.

---

## üèóÔ∏è Architecture

1.  **User Interaction**: The user selects a diagram type and enters a prompt in the **React Frontend**.
2.  **API Request**: The frontend sends a secure POST request to the **FastAPI Backend**.
3.  **AI Processing**:
    *   The backend constructs a specialized prompt based on the selected diagram type (using Few-Shot Prompting).
    *   It sends this prompt to the local **Ollama** instance running LLaMA 3.1.
4.  **Code Cleaning**: The backend receives the raw AI response, cleans it using Regex (fixing syntax errors, removing markdown), and ensures it's valid Mermaid code.
5.  **Response**: The clean code is sent back to the frontend.
6.  **Rendering**: The frontend renders the code using **Mermaid.js**.
7.  **Storage**: The diagram is asynchronously saved to **MongoDB** linked to the user's account.

## üöÄ How to Run

### Prerequisites
*   **Node.js** (v16+)
*   **Python** (v3.11+)
*   **MongoDB** (Running locally on port 27017)
*   **Ollama** (Installed and running)

### Step 1: Start Infrastructure
1.  Start MongoDB: `mongod`
2.  Start Ollama: `ollama serve`
3.  Pull the AI Model: `ollama pull llama3.1`

### Step 2: Start Backend
```bash
cd backend
# Install dependencies
pip install -r requirements.txt
# Run server
python -m app.main
```
*Server will run at: `http://localhost:8000`*

### Step 3: Start Frontend
```bash
cd frontend
# Install dependencies
npm install
# Run app
npm start
```
*App will open at: `http://localhost:3000`*
